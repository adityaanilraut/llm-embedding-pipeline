Incremental Embedding Updates

Re-embedding an entire corpus on every ingestion run is wasteful when only a handful of files changed. A practical incremental strategy:

  1. Hash each source file with SHA-256.
  2. Store (path, hash, mtime) in a tiny SQLite table.
  3. On the next run, only re-parse files whose hash changed.
  4. Within a changed file, hash each chunk; unchanged chunks reuse cached embeddings.
  5. Deleted files are tombstoned: their chunks are removed from the vector store by doc_id.

This keeps the embedding compute proportional to the change in the corpus, not its total size — critical when documents update frequently or the corpus grows into the millions.
